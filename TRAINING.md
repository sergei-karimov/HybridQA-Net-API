## 1. Подготовка данных

### Структура директорий

```
data/
├── train/
│   ├── images/
│   │   ├── product_001.jpg
│   │   ├── product_002.jpg
│   │   └── ...
│   └── annotations.json
└── val/
    ├── images/
    │   ├── product_101.jpg
    │   └── ...
    └── annotations.json
```

### Формат annotations.json

```json
[
  {
    "image":    "images/product_001.jpg",
    "query":    "На упаковке должно быть указано: Хранить при t < +25°C",
    "document": "ГОСТ 12345-2020. Раздел 4.3: Условия хранения обязательно указываются...",
    "label":    1
  },
  {
    "image":    "images/product_002.jpg",
    "query":    "На упаковке должно быть указано: Хранить при t < +25°C",
    "document": "ГОСТ 12345-2020. Раздел 4.3: Условия хранения обязательно указываются...",
    "label":    0
  }
]
```

**Поля:**

| Поле | Тип | Описание |
|------|-----|----------|
| `image` | string | Путь к изображению **относительно** папки split (`train/` или `val/`) |
| `query` | string | Что проверяем — формулируется как требование стандарта |
| `document` | string | Текст стандарта / ГОСТ / ТУ (до 512 символов эффективно) |
| `label` | int | `1` = соответствует стандарту, `0` = не соответствует |

### Требования к изображениям

| Параметр | Рекомендация |
|----------|-------------|
| Формат | JPEG или PNG |
| Разрешение | минимум 224×224 px, оптимально 400–800 px по меньшей стороне |
| Размер файла | до 20 МБ |
| Фокус | этикетка должна занимать >50% кадра |
| Освещение | равномерное, без сильных бликов |

### Сколько изображений нужно

| Режим обучения | Минимум | Рекомендуется | Ожидаемая accuracy |
|----------------|---------|---------------|-------------------|
| Только decision head (`freeze_backbone=True`) | 50 | 200+ | 65–75% |
| Fine-tune всей сети (`freeze_backbone=False`) | 150 | 500+ | 75–90% |
| Продакшн-качество | — | 1000+ | 90%+ |

**Важно:** соблюдайте **баланс классов** — примерно 50% label=1 и 50% label=0.
Допустимый дисбаланс: не более 70/30. При большем — используйте class weights (уже реализовано автоматически).

### Разделение train/val

- **train**: 80% данных
- **val**: 20% данных
- val выборка должна содержать **новые изображения**, не использованные при обучении

## 2. Генерация синтетических данных

Если реальных данных нет — используйте генератор для создания синтетических этикеток.

### Быстрый старт

```bash
# 300 изображений, проверка температурной метки
python tools/generate_dataset.py \
  --output data \
  --count 300 \
  --val-ratio 0.2 \
  --mark temperature
```

### Параметры генератора

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `--output` | `data` | Директория для сохранения |
| `--count` | `200` | Общее число изображений (train + val) |
| `--val-ratio` | `0.2` | Доля валидационной выборки |
| `--mark` | `temperature` | Что проверяем |

**Доступные признаки (`--mark`):**

| Значение | Проверяемое требование |
|----------|----------------------|
| `temperature` | Хранить при t < +25°C |
| `humidity` | Влажность не более 75% |
| `light` | Беречь от прямых солнечных лучей |
| `children` | Хранить в недоступном для детей месте |

### Ограничения синтетических данных

Синтетика хороша для начального обучения и проверки pipeline.
Для продакшна необходимо **дообучение на реальных фотографиях**.
Рекомендуемая стратегия:
1. Обучите на 300–500 синтетических изображений → получите базовую модель
2. Соберите 100–200 реальных фото → дообучите поверх

## 3. Использование реальных данных

### Разметка (аннотирование)

Для каждого изображения нужно:
1. Определить `query` — требование, которое проверяем
2. Добавить `document` — текст стандарта (или его фрагмент)
3. Проставить `label`:
   - `1` — требование выполнено (нужный элемент присутствует)
   - `0` — требование не выполнено (элемент отсутствует или некорректен)

### Пример скрипта разметки

```python
import json
from pathlib import Path

# Заполните вручную или через инструмент разметки (LabelStudio, CVAT)
annotations = []

for img_path in Path("data/train/images").glob("*.jpg"):
    annotations.append({
        "image":    f"images/{img_path.name}",
        "query":    "На упаковке должно быть указано: Хранить при t < +25°C",
        "document": "ГОСТ 12345-2020. Условия хранения: температура не выше +25°C.",
        "label":    1,  # проставьте вручную: 1 или 0
    })

with open("data/train/annotations.json", "w", encoding="utf-8") as f:
    json.dump(annotations, f, ensure_ascii=False, indent=2)
```

### Советы по съёмке реальных этикеток

- Снимайте при **естественном или рассеянном** свете
- Держите камеру **параллельно** этикетке (без сильного угла)
- Включайте в датасет **разные условия**: мятые, влажные, частично повреждённые этикетки
- Снимайте этикетки **с разного расстояния**: крупный план и общий вид

## 4. Запуск обучения

### Стратегия двухэтапного обучения (рекомендуется)

#### Этап 1 — Быстрое обучение головы (5–10 минут)

```python
from src.pipeline import HybridQANet
from training.trainer import Trainer
from training.dataset import QADataset

system = HybridQANet(config_path="configs/config.yaml")

config = system.config.get("training", {})
config["epochs"] = 20
config["batch_size"] = 8
config["learning_rate"] = 1e-4

trainer = Trainer(system, config)
history = trainer.train(
    train_dataset=QADataset("data", split="train", augment=True),
    val_dataset=QADataset("data", split="val"),
    freeze_backbone=True,   # backbone заморожен — быстро, мало памяти
)
```

Когда остановить: val accuracy перестала расти 3–5 эпох подряд.

#### Этап 2 — Fine-tune всей сети

```python
system = HybridQANet(config_path="configs/config.yaml")

# Загружаем лучший чекпоинт из этапа 1
system.load_checkpoint("checkpoints/best.pt")

config = system.config.get("training", {})
config["epochs"] = 30
config["batch_size"] = 4    # меньше batch — больше памяти под градиенты
config["learning_rate"] = 2e-5   # меньший lr чтобы не сломать backbone

trainer = Trainer(system, config)
history = trainer.train(
    train_dataset=QADataset("data", split="train", augment=True),
    val_dataset=QADataset("data", split="val"),
    freeze_backbone=False,  # обучаем всю сеть
)
```

### Мониторинг в реальном времени

```python
import json

with open("training_history.json", "w") as f:

```

### Параметры configs/config.yaml

```yaml
training:
  batch_size: 8          # Уменьшите до 4 если нет памяти
  learning_rate: 2.0e-5  # Для fine-tune; 1e-4 для head-only
  epochs: 30
  warmup_steps: 100      # Шаги плавного старта lr
  weight_decay: 0.01     # L2 регуляризация
  gradient_clip: 1.0     # Ограничение градиентов
  eval_every: 5          # Валидация каждые N эпох
  save_top_k: 3          # Хранить топ-N чекпоинтов
  checkpoint_dir: "./checkpoints"
```

## 5. Оценка результатов

### На что смотреть в логах

```
Эпоха 10/30 | Loss: 0.4821 | LR: 1.85e-05 | Время: 5.3с
  Валидация → Loss: 0.5103 | Accuracy: 0.7500
```

| Метрика | Хорошо | Плохо | Что делать |
|---------|--------|-------|------------|
| Train Loss | стабильно падает | растёт или не меняется | уменьшите lr |
| Val Loss | близко к train loss | намного выше train | переобучение — добавьте данных |
| Val Accuracy | >70% | ~50% | модель не учится — см. раздел 6 |
| Разрыв train/val loss | <0.1 | >0.3 | переобучение — аугментация, dropout |

### Интерпретация accuracy

| Val Accuracy | Интерпретация |
|-------------|--------------|
| ~50% | Случайное угадывание — модель не обучается |
| 60–70% | Слабое обучение — нужно больше данных или другой lr |
| 70–80% | Приемлемо для пилота |
| 80–90% | Хорошо для задачи |
| >90% | Отлично — проверьте что нет утечки данных |

**Признаки хорошей модели:**
- Карта внимания фокусируется на тексте/элементе из запроса
- При запросе про температуру — выделяет строку с температурой

## 6. Частые проблемы

### Accuracy застряла на 50%

**Причины и решения:**

| Причина | Решение |
|---------|---------|
| Слишком мало данных (<100) | Добавьте данных или используйте генератор |
| `freeze_backbone=True` + маленький head | Попробуйте `freeze_backbone=False` |
| Слишком высокий lr | Уменьшите до `2e-5` |
| Дисбаланс классов | Убедитесь что label=0 и label=1 примерно поровну |
| Все query одинаковые | Разнообразьте формулировки запросов |

### Val Loss растёт при падающем Train Loss (переобучение)

```yaml
# configs/config.yaml
training:
  weight_decay: 0.05    # увеличьте с 0.01
```

Или добавьте данных / используйте аугментацию:
```python
QADataset("data", split="train", augment=True)  # augment=True обязательно
```

### Out of Memory

```yaml
training:
  batch_size: 4   # уменьшите с 8 до 4 или до 2
```

### Обучение идёт очень медленно

На CPU каждая эпоха занимает ~5–10 сек при batch=8 и 300 изображениях.
При наличии GPU скорость возрастает в 10–50 раз.

Проверить устройство:
```python
import torch
print(torch.cuda.is_available())   # True если есть CUDA GPU
```

## Рекомендуемый порядок действий

```
1. Сгенерируйте 300 синтетических изображений
   python tools/generate_dataset.py --count 300

2. Запустите обучение (этап 1, 20 эпох, freeze_backbone=True)
   → Смотрите на val accuracy: должна расти выше 60%

3. Если accuracy растёт — запустите этап 2 (freeze_backbone=False)
   → Целевая val accuracy: >75%

4. Проверьте модель на реальных фото через API
   → Смотрите на карту внимания: должна фокусироваться на нужной зоне

5. Соберите реальные данные и дообучите
   → 100–200 реальных фото дают значительный прирост качества
```
